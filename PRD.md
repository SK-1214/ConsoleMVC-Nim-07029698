# PRD: 반도체 시료 생산 주문 관리 시스템

## 1. 배경 및 목적

가상의 반도체 회사 **S-Semi**는 다양한 종류의 반도체 시료(Sample)를 생산하여 연구소, 팹리스(Fabless) 업체, 대학 연구실 등의 고객에게 납품한다. 시료는 주문이 들어오면 웨이퍼 공정 설비를 통해 제작되고, 검수를 거쳐 고객에게 출고된다.

기존에는 엑셀과 메모장으로 주문을 관리하여 실수가 잦고, 재고와 공정 현황을 한눈에 파악하기 어려웠다. 이를 해결하기 위해 **콘솔 기반**의 "반도체 시료 생산 주문 관리 시스템"을 개발한다.

---

## 2. 시스템 개요

- **생산 라인**: 공장에서 시료 하나를 생산하는 설비 흐름. 하나의 생산 라인은 시료를 하나씩 생산하며, 주문이 들어온 시료에 대해서만 생산
- **운영 방식**: 담당자가 직접 명령을 입력하여 시료를 등록하고 주문을 처리

---

## 3. 주문 상태 흐름

모든 주문은 아래 5가지 상태 중 하나를 가진다. `REJECTED`는 정상 흐름 외의 상태로 모니터링에서 제외한다.

| 상태 | 의미 |
|------|------|
| `RESERVED` | 주문 접수 |
| `REJECTED` | 주문 거절 |
| `PRODUCING` | 주문 승인 완료 및 재고 부족으로 생산 중 |
| `CONFIRMED` | 주문 승인 완료 및 출고 대기 중 |
| `RELEASE` | 출고 완료 |

**상태 전이 규칙**

```
RESERVED ──[승인, 재고 충분]──→ CONFIRMED ──[출고]──→ RELEASE
RESERVED ──[승인, 재고 부족]──→ PRODUCING ──[생산 완료]──→ CONFIRMED ──[출고]──→ RELEASE
RESERVED ──[거절]──────────→ REJECTED
```

## 4. 기능 명세

### 4.1 메인 메뉴

시작 시 기능별 선택 화면을 표시하며, 전체 시료에 대한 요약 정보를 함께 확인할 수 있다.

| 메뉴 | 설명 |
|------|------|
| 시료 관리 | 새로운 시료 등록, 목록 조회, 이름 검색 |
| 주문 (접수/승인/거절) | 고객 주문 접수 및 생산 라인 담당자의 승인·거절 처리 |
| 모니터링 | 상태별 주문 수 및 시료별 재고 현황 확인 |
| 출고 처리 | `CONFIRMED` 상태 주문에 대해 출고 실행 |
| 생산 라인 | 현재 생산 중인 시료 및 대기 중인 생산 큐 확인 |

---

### 4.2 시료 관리

시료(Sample)는 시스템의 가장 기본이 되는 단위이며, 시스템에 등록된 시료만 주문 가능하다.

> **수율 정의**: 정상적인 시료 수 / 총 생산 시료 수  
> 예) 100개 생산 중 정상 물품 90개 → 수율 = 0.9

#### 4.2.1 시료 등록

새로운 시료를 시스템에 추가한다.

**입력 속성**

| 속성 | 설명 |
|------|------|
| 시료 ID | 고유 식별자 |
| 이름 | 시료명 |
| 평균 생산 시간 | 시료 1개 생산에 걸리는 평균 시간 |
| 수율 | 0.0 ~ 1.0 사이의 실수 (예: 0.9) |

#### 4.2.2 시료 조회

등록된 모든 시료 목록을 확인한다. 현재 재고 수량도 함께 표시한다.

#### 4.2.3 시료 검색

이름 등 속성으로 특정 시료를 검색한다.

---

### 4.3 시료 주문 (접수)

고객이 시료를 요청하면 주문 담당자가 주문을 생성한다.

#### 4.3.1 시료 예약

고객이 원하는 시료와 수량을 주문한다. 주문 생성 시 상태는 `RESERVED`로 설정된다.

**입력값**

| 필드 | 설명 |
|------|------|
| 시료 ID | 주문할 시료의 식별자 |
| 고객명 | 주문 고객 이름 |
| 주문 수량 | 요청 수량 |

---

### 4.4 주문 승인 / 거절

접수된 주문(`RESERVED`) 목록을 확인하고, 특정 주문을 승인하거나 거절한다.

#### 4.4.1 접수된 주문 목록 조회

`RESERVED` 상태의 주문 목록을 표시한다.

#### 4.4.2 주문 승인

접수된 특정 주문에 대해 승인 처리한다. 승인 시 재고 상황에 따라 **2가지 방식**으로 자동 처리된다.

| 조건 | 처리 |
|------|------|
| 재고 충분 | 주문 상태를 즉시 `CONFIRMED`로 전환 |
| 재고 부족 | 생산 라인에 자동 등록, 주문 상태를 `PRODUCING`으로 전환 |

#### 4.4.3 주문 거절

접수된 특정 주문에 대해 거절 처리한다. 즉시 `REJECTED` 상태로 전환한다.

---

### 4.5 모니터링

담당자가 현재 시스템의 상태를 한눈에 파악할 수 있도록 주문 및 재고 정보를 표시한다.

#### 4.5.1 주문량 확인

현재 상태별 주문 목록을 확인한다.

- 표시 대상 상태: `RESERVED`, `CONFIRMED`, `PRODUCING`, `RELEASE`
- `REJECTED`는 유효한 주문이 아니므로 제외

#### 4.5.2 재고량 확인

각 시료별 현재 재고 수량을 확인하며, 주문 대비 재고 수준에 따른 상태를 표기한다.

| 상태 | 조건 |
|------|------|
| 여유 | 주문 대비 재고 충분 |
| 부족 | 주문 대비 재고 수량 부족 |
| 고갈 | 재고 수량이 0인 상태 |

---

### 4.6 생산 라인

생산 라인의 현황을 표시하고, 주문량에 대한 부족분을 생산한다. 수율 및 오차를 고려하여 실제 생산량을 결정한다.

#### 4.6.1 생산량 계산 공식

```
실생산량 = ceil(부족분 / (수율 × 0.9))
총생산시간 = 평균생산시간 × 실생산량
```

#### 4.6.2 생산 완료 처리

생산이 완료되면 해당 주문의 상태를 `PRODUCING` → `CONFIRMED`로 변경한다.

#### 4.6.3 생산 현황 표기

현재 생산 중인 시료에 대한 정보를 표시한다. (예: 주문 정보, 현재까지의 생산량 등)

#### 4.6.4 대기 주문 확인

생산 큐(Queue)에 대기 중인 주문 목록을 출력한다.

- **스케줄링 전략**: FIFO (First In, First Out)

---

### 4.7 출고 처리

재고가 충분한 `CONFIRMED` 상태의 주문에 대해 출고를 처리한다.

- 특정 주문에 대해 출고를 실행하면 주문 상태가 `RELEASE`로 전환된다.

## 5. 데이터 모델 요약

### Sample (시료)

| 필드 | 타입 | 설명 |
|------|------|------|
| id | string/int | 고유 식별자 |
| name | string | 시료명 |
| avgProductionTime | float/int | 평균 생산 시간 |
| yield | float | 수율 (0.0 ~ 1.0) |
| stock | int | 현재 재고 수량 |

### Order (주문)

| 필드 | 타입 | 설명 |
|------|------|------|
| orderId | string/int | 주문 고유 식별자 |
| sampleId | string/int | 주문 시료 ID |
| customerName | string | 고객명 |
| quantity | int | 주문 수량 |
| status | enum | RESERVED / REJECTED / PRODUCING / CONFIRMED / RELEASE |

## 6. 비기능 요구사항

- 콘솔 기반 인터페이스 (Visual Studio C++ 프로젝트)
- 화면 구성은 자유롭게 결정
- 생산 라인은 FIFO 방식으로 순서 보장
- 생산 계산 시 `ceil` 함수를 사용하여 올림 처리
